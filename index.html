営業フォローアップを、もっと速く。

Sales Follower は、営業電話やメール送付の予定を Google カレンダー / タスク / Gmail と連携して 半自動化する小さなツールです。架電予定もメール下書きも、3クリックで終わります。

  • 📞 再架電予定 → Google カレンダーに自動登録
  • 📧 メール送付 → Google Tasks + Gmail 下書きを一括作成
  • 📋 サービスごとに雛形 → 商品別のフォローテンプレを管理
  • 📌 浮き出し小窓 → 常時最前面でながら作業
  • 🔒 データはあなたの Google アカウントとブラウザのみ (このアプリのサーバには何も送信されません)

    📖 導入方法

    1. 普通に使うだけなら

    1. ヘッダの「Google にログイン」をクリック
    2. 同意画面で カレンダー / Gmail / タスク へのアクセスを許可
    3. 「⚙ 各種設定」→「📋 サービス管理」で雛形を登録
    4. フォームから架電 / メール送付を登録 → カレンダー・Tasks・Gmail に反映

    2. 自分の Google Cloud プロジェクトで動かす (任意)

    組織のセキュリティポリシーなどで「自分のプロジェクトでしか OAuth を通したくない」場合、 以下の手順で各自の Client ID / API キーを使えます。

    1. Google Cloud Console でプロジェクト作成
    2. 「APIとサービス」→「ライブラリ」で以下の API を すべて有効化:
      • Google Calendar API (Meet リンク自動発行はこの API に含まれます — 別個の有効化は不要)
      • Gmail API
      • Google Tasks API
    3. 「OAuth 同意画面」を構成:
      • User Type: 外部 (テスト)
      • スコープに calendar.events / gmail.compose / tasks を追加
      • テストユーザに自分の Gmail アドレスを追加
    4. 「認証情報」→ OAuth クライアント ID 作成 (種類: ウェブアプリケーション)
      • 承認済み JavaScript 生成元: 利用中のアプリ URL (例: http://localhost:5173)
    5. 同画面で API キー を発行
    6. このアプリの「⚙ 各種設定」→「🔑 Google 認証情報」に Client ID と API キーを貼り付け → 保存
    7. 再ログインして同意画面で全権限にチェック

    3. 自分のサーバ / ホスティングにデプロイする場合

    1. このプロジェクト一式 (HTML/JS/CSS) を任意の静的ホスティング (Vercel, Netlify, GitHub Pages 等) にアップロード
    2. Google Cloud Console の OAuth クライアントで 承認済み JavaScript 生成元 にデプロイ先 URL を追加
    3. HTTPS が必須 (Google API が HTTP では拒否)
    4. 正式公開する場合は OAuth 同意画面の 公開 ボタン + ブランド検証手続き(数日)が必要

    4. 使えるスコープと必要な権限

    スコープ用途影響範囲
    calendar.events再架電予定・Meet 付き商談予定の作成・読み込みカレンダー予定のみ。設定は変更不可
    gmail.composeメール下書きの作成 (Meet URL の自動転記含む)下書きのみ作成。送信は行いません
    tasks資料送付タスクの作成・完了マークGoogle Tasks のみ

    5. サービス管理 (アクション選択)

    「⚙ 各種設定」→「📋 サービス管理」で各サービスごとに以下 3 つのアクションを ON/OFF できます。

    • 📧 Gmail 下書き作成 — 雛形を変数置換して下書きに保存
    • 📋 Google Tasks 登録 — 「【資料送付】会社名」でチェックボックス付きタスクを追加
    • 🎥 Google Meet 付き予定 — 日時指定 + Meet リンク自動発行された予定をカレンダーに登録。 Meet URL は雛形の {MeetURL} または下書き本文末尾に自動転記

    6. うまくいかないとき

    • 「access_blocked / app not verified」: OAuth 同意画面の「テストユーザ」に自分のメールを追加
    • 「Calendar / Gmail / Tasks API not enabled」: Cloud Console で該当 API を有効化
    • Meet リンクが付かない: Calendar API のみで動作。同 API が無効でないか確認
    • 401 が出続ける: 一度ログアウト → 再ログイン
    • 下書きができない: 同意画面で「下書き作成」のチェックが ON か確認

    上記で解決しない場合はお気軽にどうぞ

    🔒 プライバシーポリシー

    最終更新: 2026年5月20日

    1. はじめに

    Sales Follower (以下「本アプリ」) は、Google API を介して利用者の Google カレンダー / Gmail / Tasks を操作するブラウザ上のツールです。本アプリの運営者は、利用者の入力した 情報を一切収集しません。

    2. 収集する情報

    本アプリは利用者の情報を 当方のサーバへ送信・保存しません。 以下はすべて利用者のブラウザ内 (localStorage) に保存されます:

    • サービス雛形 (件名/本文)
    • メール署名
    • 使用中のサービス ID
    • (任意) 上書きした Google Client ID / API キー
    • Google API の同期トークン・直近のカレンダーイベント (キャッシュ)

    これらはブラウザを変える/履歴を消去すると失われ、復元できません。

    3. Google ユーザーデータの扱い

    本アプリは利用者の許諾に基づき、以下のスコープで Google API を呼び出します:

    • https://www.googleapis.com/auth/calendar.events — 予定の作成・取得
    • https://www.googleapis.com/auth/gmail.compose — 下書きの作成 (送信はしない)
    • https://www.googleapis.com/auth/tasks — タスクの作成・完了マーク

    取得した情報は その場の表示・ローカルキャッシュのみに使用し、第三者への提供や 広告利用、機械学習目的の解析は 一切行いません。

    Google API から取得したユーザーデータの権限は、利用者が Google アカウントの「サードパーティのアクセス」設定 からいつでも取り消せます。

    4. Cookie / トラッキング

    本アプリは独自の Cookie や追跡 ID を使用しません。 ただし Google の認証ライブラリ (gapi, GIS) は OAuth フロー上 Google ドメインの Cookie を使用します。

    5. 第三者サービス

    • Google LLC (Calendar / Gmail / Tasks / Identity Services) — OAuth とデータ操作のため

    6. データの削除

    ブラウザのストレージを削除するか、本アプリ「⚙ 各種設定」内のリセット機能で 保存内容を消去できます。Google 側に作成したデータ (予定/下書き/タスク) は Google 側で個別に削除してください。

    7. 連絡先

    本ポリシーに関する問い合わせ: koki.ikemoto@neo-career.co.jp

    8. 変更

    本ポリシーは予告なく更新されることがあります。重要な変更がある場合は本アプリ上で告知します。
